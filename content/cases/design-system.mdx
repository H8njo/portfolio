사내 디자인 시스템을 맡았다. 처음엔 컴포넌트만 잘 만들면 되는 줄 알았다. 막상 해보니 정작 일의 절반은 여러 사람이 그 위에서 안 헷갈리게 굴러가게 하는 쪽이었다.

여러 프론트엔드 제품이 같은 UI·접근성을 공유해야 했다. React 컴포넌트 라이브러리를 Radix primitive + Tailwind + CVA 위에 올리고, Storybook으로 문서화해 사내 패키지로 배포하는 구조다. 나는 초기 인프라를 깔고, 첫 컴포넌트인 Button으로 패턴을 잡고, 아이콘 파이프라인을 자동화했고, 최근엔 Select를 두 계층으로 갈아엎었다. 하다 보니 코드오너 겸 릴리스 매니저가 돼 있었다.

## 문제

문제 자체는 개별 화면 작업이랑 결이 좀 달랐다. 여기서 컨벤션 하나 잘못 정하면 그게 소비 프로젝트마다 그대로 복제된다. 그래서 사소한 것도 함부로 못 정했다.

아팠던 데는 크게 세 군데.

첫째, 기존 `Select`가 Radix Select 위에 multi-select를 흉내 내고 있었다. Radix Select는 본질적으로 단일 값이다. 그걸로 multi를 만들겠다고 `value`를 빈 문자열로 비우고 `shouldKeepOpenRef`로 자동 닫힘을 가로채는 우회를 쓰고 있었다. 컴포넌트가 정작 뭐가 선택됐는지를 모른다. 진짜 선택 집합도, 체크박스도, `aria-checked`도 없었다.

둘째, 아이콘이 수작업이었다. 디자이너가 Figma에서 뽑은 SVG를 누가 손으로 컴포넌트로 옮기고 barrel에 export를 한 줄씩 추가하는 식. 160개쯤 되니까 이건 일이 아니라 사고 대기다.

셋째, 손으로 만든 컴포넌트는 폴더 구조가 매번 미묘하게 달랐다. 누구는 barrel을 만들고 누구는 안 만들고, stories가 있는 것과 없는 게 섞여 있었다.

## 트레이드오프 결정

Button 초기 버전에 솔직히 좀 욕심을 부렸다. `isLoading` prop을 두는 것까지는 좋은데, async `onClick`이 promise를 반환하면 그걸 가로채서 로딩 상태를 알아서 켜고 꺼주게 만들었다. 데모로는 그럴듯했다.

근데 라이브러리 primitive로는 그냥 틀린 설계였다. 컴포넌트가 stateful해지고, 핸들러가 promise를 반환할 때만 동작하고, async 생명주기 주도권을 호출자한테서 뺏어온다. (한참 쓰다가 깨달았다.) 결국 controlled `isLoading`으로 되돌렸다. Button은 보여주기만 하고 상태는 소비자가 쥔다. primitive에서는 마법보다 예측 가능한 쪽이 거의 항상 맞더라.

Select도 같은 자리에서 같은 고민을 했다. single이랑 multi를 한 엔진에 억지로 욱여넣지 않고, **API는 통합하되 내부는 두 엔진으로 분기**시켰다. single은 Radix Select를 그대로 래핑해서 폼 바인딩(`name`/`required`/hidden input/자동 닫힘)을 공짜로 가져온다. multi는 Radix Popover 위에 선택 모델·키보드 내비·체크박스를 직접 짰다. 소비자가 쓰는 JSX는 양쪽이 똑같다.

```ts
type SelectV2SingleProps = Common & { multiple?: false; value?: string;  onValueChange?: (v: string)   => void }
type SelectV2MultiProps  = Common & { multiple: true;  value: string[];  onValueChange:  (v: string[]) => void }
export type SelectV2Props = SelectV2SingleProps | SelectV2MultiProps
```

`multiple`을 주면 컴파일러가 `value: string[]`랑 배열 콜백을 요구하고, 빼면 `string` 버전을 강제한다. 잘못 쓰면 런타임 가기 전에 타입 체크에서 막힌다.

## 접근

아이콘은 코드젠으로 뒤집었다. 디자이너가 `assets/icons/design-system/`에 SVG를 드롭하고 `pnpm generate`를 돌리면, plop이 디렉토리를 스캔해서 barrel을 통째로 다시 만든다. 파일을 지우면 export도 같이 사라진다. 폴더가 곧 진실인 셈. Figma export preset에서 `Black => currentColor`를 켜둔 덕에 모든 아이콘이 컨테이너 CSS `color`를 상속해서, per-icon variant 없이 테마잉이 된다.

근데 여기서 빌드가 두 번 죽었다.

처음엔 `?react` 한 줄이면 끝날 줄 알았다. barrel이 `'...Alert_1px.svg?react'`로 import하는데, Storybook(Vite)에선 멀쩡하고 라이브러리 빌드(Rollup)만 죽었다. 원인 찾는 데 한참 헤맸다. 알고 보니 `?react`는 vite-plugin-svgr가 만든 컨벤션이라 Vite에서만 의미가 있는 거였다. Rollup의 `@svgr/rollup`은 그 쿼리를 못 읽고, `Alert_1px.svg?react`라는 디스크에 없는 파일을 찾다가 resolution에서 질식했다. barrel을 두 벌로 가르는 대신, 쿼리를 떼고 실제 경로로 넘겨주는 작은 Rollup 플러그인을 끼웠다.

```js
{
  name: 'svg-query-resolver',
  resolveId(source, importer) {
    if (source.endsWith('?react')) {
      const actualPath = source.replace('?react', '')
      return path.resolve(path.dirname(importer), actualPath)
    }
    return null
  },
}
```

결과적으로 단일 barrel이 Vite(네이티브)랑 Rollup(투명 rewrite) 양쪽에서 돈다. 개발 toolchain이랑 배포 toolchain이 다를 땐 둘 중 하나에 맞춰 소스를 갈라치는 것보다 얇은 어댑터 하나 끼우는 게 나았다.

두 번째 죽음은 더 음흉했다. 빌드는 통과하는데 화면에서 아이콘이 잘렸다. `className="h-7 w-7"`로 키우면 스케일이 안 되고 클리핑된다. 범인은 SVGO였다. `width`/`height`랑 `viewBox`가 둘 다 있으면 `removeViewBox`가 viewBox를 redundant로 보고 지워버린다. viewBox가 없어지면 좌표계가 사라져서 CSS 스케일이 path를 잘라먹는다. 이것도 한참 째려봤다. Figma를 viewBox-only로 다시 뽑고, 빌드 쪽 SVGO 설정에 `removeViewBox: false`로 못을 박았다. 소스랑 빌드 두 층 다 막아둬야 입력이 흔들려도 안 터진다.

`?react` 쪽은 타입 체크만으론 안 잡혔다(ambient `*.svg` 선언이 그냥 통과시킨다). 그래서 push 전에 type-check + 실제 Rollup 빌드를 강제하는 게이트를 걸었다. 같은 부류 실패는 이제 내 로컬을 못 벗어난다.

폴더 구조 드리프트는 plop 제너레이터로 막았다. 프롬프트 하나로 컴포넌트·stories·barrel 3-파일 폴더를 컨벤션대로 찍어낸다. 이름은 정규식으로 PascalCase를 강제하고, stories는 `autodocs`가 미리 배선돼 있어서 만들면 바로 문서가 된다. 아이콘 코드젠도 결국 이 제너레이터의 `icon` 분기로 합쳐버렸다.

## 결과

Select는 multi-select가 우회가 아니라 일급 모델이 됐다. 제일 마음에 드는 건 mirror mode다. trigger를 최장 옵션 너비에 맞추고 싶은데, 옵션이 열리기 전엔 portal 안에 있어서 측정이 안 된다. 그래서 숨긴 DOM mirror를 trigger랑 같은 grid 셀에 겹쳐놓고, `inline-grid w-fit`이 셀을 mirror에 맞추게 한 다음 trigger가 그 너비를 입게 했다. ResizeObserver도 `useLayoutEffect`도 reflow 루프도 없는 순수 CSS 측정이다. (이거 떠올리는 데 며칠 걸렸다.)

이게 보상받는 건 소비자 쪽이다. 교재→단원 tree-select에 탭까지 붙은 복잡한 selectbox를, trigger 스타일 한 줄 다시 안 짜고 조립할 수 있었다. multi context가 중첩 깊이랑 상관없이 전파되니까, grid랑 Tabs 안에 묻힌 `SelectV2.Item`도 알아서 체크박스 붙고 토글된다. 디자인 시스템 떼낸 이유가 사실 이거였다.

아이콘은 export 보일러플레이트가 통째로 사라졌다. 디자이너가 폴더에 드롭하면 타입 있는 아이콘 컴포넌트가 나오고 Storybook 갤러리에 바로 뜬다.

역할로 보면, 설정상 모든 PR의 코드오너라 [확인]개 PR을 리뷰·머지했고 0.0.6·0.0.16 릴리스를 직접 cut했다. PR 템플릿(버그는 AS-IS/TO-BE 표 강제)이랑 리뷰어 로테이션 정책을 만들고, commit/pr/ship 컨벤션을 Claude Code 스킬로 박아서 누가 와도 같은 규율로 굴러가게 했다.
