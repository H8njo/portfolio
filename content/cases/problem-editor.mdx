단어를 오른쪽에서 왼쪽으로 드래그했더니 생략 메모가 엉뚱한 단어 사이에 찍혔다.

교육 콘텐츠 편집 도구의 본문분석 에디터를 만들 때 일이다. 영어 지문 위에 밑줄·생략표시·슬래시·메모 같은 데코레이터를 10종쯤 얹는 기능이었는데, 사용자가 문장 일부를 문자 단위로 선택하면 그 범위에 표시가 그려진다. 화면은 두 개다. 왼쪽은 지문을 실제로 렌더하는 미리보기 패널, 오른쪽은 데코레이터 속성을 편집하는 폼 패널. 둘이 한 몸처럼 움직여야 했다. 폼에서 색을 바꾸면 미리보기가 즉시 따라오고, 미리보기에서 범위를 다시 그으면 폼이 그 범위를 받아 든다.

처음엔 선택한 인덱스 그대로 저장하면 되는 줄 알았다. 아니었다.

## 트레이드오프 결정

데코레이터마다 좌표를 어떻게 잡을지 두 갈래였다. 하나는 선택한 모든 단어 인덱스를 펼쳐서(`[2,3,4,5]`) 통째로 저장하는 방식, 다른 하나는 양 끝점 두 개(`[min, max]`)만 저장하는 방식.

밑줄처럼 범위 전체를 칠하는 데코레이터는 전체 배열이 편했다. 그런데 슬래시(`/`)와 생략 메모는 양 끝점만 쓴다. 슬래시는 시작 단어 뒤에 한 번 긋고, 생략 메모는 시작과 끝 두 지점만 본다. 여기에 전체 배열을 욱여넣으면, 소비하는 쪽이 `[0]`과 `[1]`만 읽으니까 `[2,3,4,5]`가 조용히 `[2,3]`으로 잘려 버린다. 타입은 둘 다 `number[]`라 컴파일러도 안 잡아준다.

그래서 데코레이터를 **끝점-페어 타입과 전체-배열 타입으로 명확히 갈랐다.** 의미가 다르면 타입이 같아 보여도 분기를 분명히 해야 한다는 게 이 작업에서 제일 비싸게 배운 것이다.

## 접근

전체 편집 흐름은 큐 기반 상태머신으로 잡았다. 사용자가 미리보기에서 범위를 선택하면 그 선택이 큐에 항목으로 쌓이고, 폼에서 데코레이터 종류와 속성을 확정하면 `applyRangeToQueueItem`이 그 항목을 실제 좌표로 변환해 미리보기에 반영한다. 선택 → 편집 → 확정이 한 방향으로 흐른다.

큐가 떠 있는 동안엔 **모드 락**을 걸었다. 한 데코레이터를 편집하는 중에 사용자가 다른 문장 섹션을 건드리면 두 선택이 엉켜 좌표 계산이 무너진다. 그래서 작업 중인 섹션 밖의 선택은 막았다. 편집이 끝나야 락이 풀린다.

역방향 선택 버그는 저장 직전 단 한 지점에서 잡았다. 우→좌 드래그는 `startIndex > endIndex` 상태가 되는데, 좌표를 그리는 하류 코드는 죄다 "낮은 인덱스가 먼저"를 암묵적으로 가정하고 있었다. 그래서 끝점-페어를 저장하는 그 한 줄에서 정규화했다.

```ts
// applyRangeToQueueItem.ts
if (d.design === 'slash' || (d.design === 'memo' && d.type === 'omit')) {
  const { startIndex: s, endIndex: e } = input.primary
  return s <= e ? [s, e] : [e, s] // 항상 [min, max]
}
```

여기서 한 번 정규화하니까 슬래시도, 생략 메모도, 그 아래 모든 렌더 코드가 드래그 방향을 더는 신경 쓰지 않아도 됐다.

좌표가 흔들리는 또 다른 경로도 있었다. 폼 패널이 늘었다 줄거나 화면이 작아지면 단어 줄바꿈이 바뀌어 데코레이터가 옛 줄바꿈 기준으로 어긋났다. 줄바꿈을 바꾸는 건 측정 컨테이너 자신이 아니라 그 부모 레이아웃이었다. `ResizeObserver`로 컨테이너만 보고 있었으니 부모가 줄어도 트리거가 안 됐다. 부모까지 같이 관찰하고, 비동기 누락 대비로 `window resize` fallback도 붙였다.

마지막은 렌더링 잔상. 밑줄에 dim(opacity 40%) transition을 걸었더니 일부 브라우저에서 absolute border가 sub-pixel 잔상으로 남았다. Safari에서만 재현돼서 원인 찾는 데 한참 걸렸는데, absolute border + opacity transition 조합이 별도 합성 레이어를 안 만드는 게 문제였다. `transform-gpu`(`translateZ(0)`)와 `will-change: opacity`로 그 요소만 GPU 레이어로 올려 강제로 깔끔히 다시 그리게 했다.

## 결과

선택 방향, 부모 리사이즈, 합성 레이어까지 좌표가 어긋날 수 있는 경로를 하나씩 닫았다. 데코레이터 10종이 미리보기와 폼 양쪽에서 같은 범위를 보고, 우→좌로 그어도 제자리에 찍힌다. 핵심은 화려한 알고리즘이 아니라, 선택 결과를 저장하는 단 한 지점에서 정규화하고 타입의 의미를 분기로 못 박은 것이었다. [확인: QA 티켓 수 / 재현 버그 감소]
